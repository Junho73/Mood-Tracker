# Mood Tracker (Mooda Clone) 🌌

> **💡 학습 목적 (Learning Purpose)**
> 본 프로젝트는 **React**를 이용한 다이내믹 UI 제어 및 상태 관리와 **Cypress**를 활용한 E2E(End-to-End) 테스트를 연습하고 학습하기 위해 제작된 토이 프로젝트입니다. 
> 특정 감정 기록 앱(Mooda)의 귀여운 캘린더 UI 감성을 모방(Clone)하여 바이브 코딩(Vibe Coding)을 실습하는 것을 목표로 하였습니다.

하루하루의 감정을 이모지 캐릭터로 기록하고, 한 달 단위로 감정 변화를 모아볼 수 있는 감성적인 감정 기록 일기장입니다.

## ✨ 주요 기능
- 📅 **캘린더 뷰**: 한 달의 감정 흐름을 한눈에 파악할 수 있는 넓은 캘린더 인터페이스.
- 😊 **이모지 감정 선택**: 직관적이고 귀여운 10가지 이모지(😊, 🤩, 🥰, 😌, 😴, 🥺, 😭, 😰, 😡, 🤒)를 활용한 감정 표현.
- 🎨 **다이내믹 Vibe**: 오늘 선택한 감정에 따라 앱의 전체적인 배경색과 분위기가 부드럽게 변환.
- 📝 **오늘의 일기**: 짧은 글과 함께 그날의 기분을 기록.

## 🛠️ 기술 스택
- **React (Vite, TypeScript)**: 컴포넌트 기반 아키텍처 및 로컬 상태 관리(Context/State) 실습.
- **CSS Modules / Variables**: 순수 CSS 변수를 활용한 다이내믹 테마 변환 실습.
- **date-fns**: 달력 렌더링에 필요한 날짜 수학 로직 구현.
- **Cypress**: 시간 고정(Time Mocking), DOM 상호작용 및 폼 유효성 검증을 포함한 E2E 테스트 자동화 구축 실습.

## 🧪 Cypress E2E 테스트 시나리오 검증 내용

프론트엔드 E2E 테스트 학습을 위해 특정 날짜를 클릭하고, 감정을 기록하고, 달력을 이동할 때 상태가 제대로 렌더링 되는지 확인하기 위한 시나리오를 구성했습니다.

### 1. 테스트 핵심 전략: Time Mocking 및 결정적(Deterministic) 렌더링
달력을 렌더링 할 때 매번 테스트 시점의 오늘 날짜가 바뀌면 테스트가 깨질 수 있습니다. 이를 방지하기 위해 `cy.clock(new Date(2026, 5, 15).getTime())`을 활용하여 시스템 시간을 항상 2026년 6월 15일로 고정하고 테스트를 진행했습니다.

### 2. 주요 테스트 시나리오 (`cypress/e2e/mood.cy.ts`)

#### 🛑 시나리오 A: 유효성 검사 (Validation)
1. 캘린더에서 날짜를 클릭하여 모달을 엽니다.
2. 기분이나 일기 내용을 적지 않고 '기록하기' 버튼을 누릅니다.
3. 상황에 맞는 에러 메시지(기분을 먼저 선택해주세요 / 오늘의 일기를 남겨보세요)가 빨간색으로 출력되는지 검증합니다.

#### 📝 시나리오 B: 감정 기록 및 캘린더 렌더링
1. 특정 날짜(15일) 셀을 클릭하여 모달을 엽니다.
2. 우울한 감정(🥺)을 선택하고 일기 내용을 작성 후 저장합니다.
3. 모달이 닫히며 캘린더의 15일 칸 중앙에 🥺 이모지가 올바르게 렌더링 되는지 확인합니다.

#### 📅 시나리오 C: 캘린더 달력 이동 시 데이터 유지 검증
1. 임의의 날짜(10일)에 화남(😡) 감정을 기록합니다.
2. 상단의 화살표 버튼을 눌러 다음 달(7월)로 이동합니다.
3. 다시 이전 달(6월)로 되돌아옵니다.
4. 로컬 상태 관리를 통해 기록해둔 화남(😡) 감정이 여전히 10일 칸에 보존되어 있는지 확인합니다.

### 3. 테스트 실행 방법

```bash
# 1. 개발 서버를 먼저 실행합니다 (별도의 터미널 창)
npm run dev

# 2. Cypress 테스트를 실행합니다 (다른 터미널 창)
# UI를 보면서 테스트하려면:
npx cypress open
# 터미널에서 헤드리스 모드로 실행하려면:
npx cypress run
```
